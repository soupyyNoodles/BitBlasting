### 1 Search in High Dimensional Space [30 marks]

In modern healthcare systems, each patient in a large hospital database can be represented as a feature vector derived from electronic health records, including clinical measurements, diagnoses, and history. Separately, we are given a set of incoming query patients (e.g., new or recently observed cases) drawn from a similar population. Identifying which existing patients in the database are most “representative” of these queries is an important step in clinical decision support and cohort discovery.

Given a database of patient representations (the base set) and a separate set of query patients, your task is to identify those database patients that best represent the query population. For each query patient, a fixed number k of its most similar database patients are retrieved. A database patient is considered more representative if it appears frequently across these retrieved sets over all queries. You must return a ranked list of the top-K most representative database patients under a strict time budget. Solutions will be evaluated against a hidden ground-truth ranking derived from this notion of representativeness.

Dataset

The provided data is organized as follows:
• data.npy: database vectors to be indexed.
• visible.npy: hidden evaluation queries used only for grading.
• visible_GT.txt: ground-truth top-K representative patients for the given query set visible.npy.

### Metric

We evaluate the submitted ranking using nDCG@K.

Ground-Truth Relevance

Let ri denote the rank of item i in the hidden ground-truth ranking. The relevance score is defined as:

rel(i) = (
K − ri + 1, if ri ≤ K,
0, otherwise.
)

---

### DCG and nDCG

For a submitted ranking (ans1, . . . , ansK), define:

DCG@K =
sum from j = 1 to K of
rel(ansj) / log2(j + 1)

Let IDCG@K be the DCG of the ideal ranking. Then:

nDCG@K = DCG@K / IDCG@K

---

### Competitive Evaluation

Submissions are evaluated under a fixed runtime budget. The solver must process the full query set and produce the final ranked list within the time limit. Performance is measured by the final nDCG@K on the hidden test queries. Then the competitive score will be computed from it similar to previous assignments.
